CivicDesk — Module 2.1 (IAM) · API Reference

Base URL: http://localhost:8081/civicDesk/iam  |  Swagger UI: /civicDesk/iam/swagger-ui.html

Conventions

  • Auth: stateless JWT (HS256). Send Authorization: Bearer <token> on protected calls. Token claims: userId, role only. Lifetime 1800 s (30 min).
  • Sliding expiry: when ≤600 s remain, a fresh token is returned on the Authorization response header (exposed via Access-Control-Expose-Headers).
  • Roles: CITIZEN, FIELD_OFFICER, DEPT_SUPERVISOR, ENGINEER, COMPLIANCE_OFFICER, ADMIN.
  • Error codes: 400 validation/bad request · 401 bad credentials / no token · 403 wrong role, wrong portal, or password not set · 404 not found · 409 duplicate email · 423 account suspended.

Endpoints

Method & pathAccessRequest bodySuccess response
POST /auth/register Public name, email, password (≥8), phone?, dateOfBirth?, gender?, nationalId?, address?, ward?, zone? 201 { "message": "Registration successful" }
POST /auth/citizen/login Public (CITIZEN only) email, password 200 { "message": "Login successful", "data": { token, userId, role, expiresIn:1800 } }
POST /auth/staff/login Public (staff only) email, password 200 { "message": "Login successful", "data": { token, userId, role, expiresIn:1800 } }
POST /auth/setPassword Public (one-time) email, newPassword (≥8) 200 { "message": "Password set successfully. Please login." }
POST /auth/logout Authenticated — 200 { "message": "Logout successful" }
GET /users/me Authenticated — 200 { "data": { userId, name, email, phone, role, departmentId, status, createdAt } }
POST /users ADMIN, DEPT_SUPERVISOR name, email, phone?, role, departmentId (ADMIN: required + must exist; SUPERVISOR: taken from JWT, body ignored) 201 { "message": "User created successfully" }
GET /users?page&size ADMIN (all) · DEPT_SUPERVISOR (own dept) — 200 { "data": { content:[...], totalElements, totalPages, currentPage } }
PUT /users/{id}/status ADMIN only status (Active | Inactive | Suspended) 200 { "message": "User status updated to {status}" }
GET /auditLogs?page&size ADMIN, COMPLIANCE_OFFICER — 200 { "data": { content:[{ auditId, userId, action, module, ipAddress, timestamp }], totalElements, totalPages, currentPage } }
GET /departments ADMIN, DEPT_SUPERVISOR — 200 { "data": [ { departmentId, name, departmentSupervisorId } ] }

Notes

  • Paths are relative to the context path /civicDesk/iam (e.g. full URL http://localhost:8081/civicDesk/iam/auth/staff/login).
  • First-login flow: admin/supervisor-created accounts have no password (isPasswordSet=false); staff login returns 403 "Please set your password before logging in" until /auth/setPassword succeeds (works once). Citizens set their password at registration.
  • Create user: ADMIN creates DEPT_SUPERVISOR only (and the department's departmentSupervisorId is set to the new user); DEPT_SUPERVISOR creates FIELD_OFFICER / ENGINEER / COMPLIANCE_OFFICER within its own department.
  • Seeded on startup: 6 departments (Infrastructure, Public Health, Licensing & Compliance, Citizen Services, Administration, Compliance & Audit) and a default ADMIN (admin@civicdesk.gov / Admin@12345, overridable via ADMIN_* env vars).
  • Audit actions: REGISTER, LOGIN, LOGOUT, CREATE_USER, UPDATE_STATUS, SET_PASSWORD, SEED_ADMIN.
