CivicDesk — Microservices Integration Guide

How each module team wires Authentication (JWT) and Audit logging into their own service.

Who this is for Teammates building a module as its own Spring Boot service (Grievance, Permit, Service Request, …). IAM issues and validates JWTs; the Audit service stores audit entries. Your service must validate the same JWTs and POST audit entries over HTTP. This guide shows exactly how.

1. Architecture & ports

Each module is its own Spring Boot app on its own port. They share nothing in memory — they talk over HTTP. The trust between them is the stateless JWT: every service holds the same signing secret, so each one validates a token locally without calling IAM on every request. 

                         ┌──────────────────────────┐
        Frontend ───────▶ │  API Gateway (optional)  │ :8080
                          └───────────┬──────────────┘
   /iam/**, /audit/**  ───▶ ┌─────────▼──────────┐
                            │ IAM + Audit service│ :8081   issues JWT · validates JWT
                            │  (one app)         │         owns users + audit_log
                            └─────────▲──────────┘
   /grievance/**  ─▶ ┌───────────────┴────┐
                     │ Grievance service  │ :8083  ──POST /audit/auditLogs (forward JWT)─▶ 8081
                     └────────────────────┘
   /permit/**     ─▶ ┌────────────────────┐
                     │ Permit service     │ :8084  ──POST /audit/auditLogs─────────────-─▶ 8081
                     └────────────────────┘
   /servicerequest/**┌────────────────────┐
              ─▶      │ ServiceRequest svc │ :8085  ──POST /audit/auditLogs─────────────-─▶ 8081
                     └────────────────────┘
ServicePortOwnsTalks to
IAM + Audit8081users, departments, audit_log—
Grievance8083grievancesAudit (8081)

Key idea IAM issues tokens (on login). Your service only validates them — using the same secret. No per-request call back to IAM. That is what keeps the services independent.

2. Repo / build structure

Recommended: a Maven multi-module build — one shared library jar + one runnable app per service.

civicdesk/                         parent pom (packaging = pom)
├── civicdesk-common/              SHARED LIBRARY (jar) — auth + response + audit-client
├── iam-service/          :8081    module/iam + module/auditlog (server side)
├── grievance-service/    :8083    module/grievance   (your service)
├── permit-service/       :8084
└── servicerequest-service/ :8085

Everything in section 3 lives in civicdesk-common. Your service adds one dependency on it instead of copy-pasting. If a shared library is not ready yet, copy the listed files into your service under the same package names.

3. Files your service needs from IAM

Bucket A — Security / JWT MUST HAVE

FileWhat it does
common/util/JwtUtil.javaValidate a token, extract userId & role. Needs the jjwt dependency.
config/JwtConfig.javaSupplies the secret + expiry to JwtUtil. Secret must equal IAM's.
…/security/JwtAuthFilter.javaReads Authorization: Bearer …, validates, puts the principal in the SecurityContext.
config/SecurityConfig.javaWires the filter, stateless sessions, anyRequest().authenticated(). You change only the permitAll() list.
common/util/SecurityContextUtil.javagetCurrentUserId() / getCurrentRole() in your controllers.
common/util/ClientIpUtil.javaResolves the caller IP (used when you log audit locally; optional otherwise).

Bucket B — Standard API contract SHOULD HAVE

FileWhat it does
common/response/ApiResponse.javaThe { message, data } envelope every endpoint returns.
common/response/PageResponse.javaPagination envelope for list endpoints.
common/exception/GlobalExceptionHandler.java + the exceptions you throwUniform 400/403/404/409 error bodies (ResourceNotFoundException, BadRequestException, ForbiddenException, …).

Bucket C — Audit integration NEEDED TO LOG

FileWhat it does
module/auditlog/client/AuditClient.javaPOSTs an audit entry to /audit/auditLogs, forwarding the JWT.
module/auditlog/dto/request/CreateAuditLogRequest.javaThe request body AuditClient builds.
module/auditlog/validation/EnumValid.java + EnumValidator.javaThe request DTO is annotated with @EnumValid, so these must be present to compile.
module/auditlog/enums/AuditAction.java + AuditModule.javaCanonical values. You add your own constants (e.g. CREATE_GRIEVANCE, GRIEVANCE).
Do NOT copy these IAM-internal classes User / Department entities & repositories, AuthController / AuthService, UserController, DepartmentController, DataSeeder, NationalIdUtil, and every server-side module/auditlog class (entity, repository, service, controller). Those belong only to IAM and the Audit service.

4. Adding AUTH to your service

Goal: every endpoint in your service is protected, and you can read who is calling.

Add the dependency (or copy Bucket A)

<dependency>
  <groupId>com.civicdesk</groupId>
  <artifactId>civicdesk-common</artifactId>
  <version>1.0.0</version>
</dependency>

No shared lib yet? Copy the Bucket A files into your service, keeping the same package names.

Set the SAME JWT secret in your application.properties

app.jwt.secret=civicdesk_hs256_secret_key_minimum_32_characters_required
app.jwt.expiry=1800000
This is the #1 cause of failures If your secret differs from IAM's by even one character, every token IAM issued will fail validation in your service with 401. The secret must be byte-for-byte identical across all services (use an env var in prod).

Adjust SecurityConfig to your own public paths

Your service has no /iam/auth/* endpoints. Replace the IAM permitAll() entries with your own public ones (often none) and keep anyRequest().authenticated().

.authorizeHttpRequests(auth -> auth
        // your public endpoints (if any), else remove these lines
        .requestMatchers("/swagger-ui/**", "/v3/api-docs/**").permitAll()
        .anyRequest().authenticated())
.addFilterBefore(jwtAuthFilter, UsernamePasswordAuthenticationFilter.class);

Read the current user/role in your controller

import com.civicdesk.common.response.ApiResponse;
import com.civicdesk.common.util.SecurityContextUtil;
import org.springframework.http.ResponseEntity;
import org.springframework.security.access.prepost.PreAuthorize;
import org.springframework.web.bind.annotation.GetMapping;

@GetMapping("/grievance/example")
@PreAuthorize("hasAnyRole('ADM','DS')")          // role codes: CIT/FO/DS/ENG/CO/ADM
public ResponseEntity<ApiResponse> example() {
    String userId = SecurityContextUtil.getCurrentUserId(); // e.g. "10000023"
    String role   = SecurityContextUtil.getCurrentRole();   // e.g. "DS"
    return ResponseEntity.ok(ApiResponse.data(service.doWork(userId, role)));
}
JWT claims you can rely on The token carries userId and role. Roles are exactly: CIT Citizen · FO Field Officer · DS Dept. Supervisor · ENG Engineer · CO Coordinator · ADM Admin. Use them verbatim in @PreAuthorize (Spring adds the ROLE_ prefix for you).

5. Adding AUDIT to your service

Goal: record who did what, in which module, from which IP by POSTing to the Audit service.

Add your action(s) and module to the enums

Edit AuditAction / AuditModule (in civicdesk-common, the single source of truth):

// AuditAction.java   →  add your verbs
CREATE_GRIEVANCE, UPDATE_GRIEVANCE, CLOSE_GRIEVANCE

// AuditModule.java   →  your module is already listed (GRIEVANCE, PERMIT, …)

Point your service at the Audit service

app.audit.base-url=http://localhost:8081/civicDesk   # → POST {base-url}/audit/auditLogs

Inject AuditClient and record after the action

import com.civicdesk.common.response.ApiResponse;
import com.civicdesk.common.util.SecurityContextUtil;
import com.civicdesk.module.auditlog.client.AuditClient;
import com.civicdesk.module.auditlog.enums.AuditAction;
import com.civicdesk.module.auditlog.enums.AuditModule;
import jakarta.servlet.http.HttpServletRequest;
import jakarta.validation.Valid;
import org.springframework.http.HttpHeaders;
import org.springframework.http.ResponseEntity;
import org.springframework.security.access.prepost.PreAuthorize;
import org.springframework.web.bind.annotation.PostMapping;
import org.springframework.web.bind.annotation.RequestBody;

// constructor-inject: private final AuditClient auditClient;
@PostMapping("/grievance")
@PreAuthorize("hasRole('CIT')")
public ResponseEntity<ApiResponse> create(
        @Valid @RequestBody CreateGrievanceRequest req,
        HttpServletRequest httpReq) {

    String userId = SecurityContextUtil.getCurrentUserId();
    grievanceService.create(req, userId);

    // forward the caller's JWT so Audit authenticates the same principal
    String authHeader = httpReq.getHeader(HttpHeaders.AUTHORIZATION);
    auditClient.record(userId, AuditAction.CREATE_GRIEVANCE, AuditModule.GRIEVANCE, authHeader);

    return ResponseEntity.status(201).body(ApiResponse.of("Created successfully", null));
}
Best-effort by design AuditClient never throws — if the Audit service is down or returns an error, it logs a warning and your main operation still succeeds. The client IP is resolved by the Audit service, so you do not send it.
What the POST endpoint expects POST /audit/auditLogs · any authenticated user · body { "userId", "action", "module" } · action/module are validated against the enums (unknown value → 400) · returns 201 with the created record.

7. Request flows

Auth flow

1. Browser → IAM (8081)        POST /iam/auth/staff/login {email, password}
                               ◀── 200 { token: <JWT signed with app.jwt.secret> }

2. Browser → YOUR svc (8083)   GET /grievance/... + Authorization: Bearer <JWT>
       │
       └─ JwtAuthFilter validates the JWT LOCALLY (same secret) → sets userId + role
          → @PreAuthorize checks the role → your controller runs

Audit flow

YOUR svc (8083)  ──POST /audit/auditLogs──▶  Audit (8081)
                   Authorization: Bearer <JWT>       │ JwtAuthFilter validates (same secret)
                   { userId, action, module }        │ action/module checked vs enums
                                                      │ IP resolved server-side
                 ◀────────── 201 Created ────────────┘ row inserted in audit_log

8. Troubleshooting

SymptomLikely causeFix
401 on every call (with a token)JWT secret differs from IAM, or token expiredMake app.jwt.secret identical to IAM; re-login for a fresh token (30-min expiry)
401 with no tokenMissing Authorization: Bearer … headerSend the header; note unknown paths also return 401 because security runs before routing
403 ForbiddenValid token but wrong role for the endpointCheck the @PreAuthorize role vs the caller's role claim
404 on audit POSTWrong app.audit.base-url or pathMust resolve to {base-url}/audit/auditLogs
Audit 400 "must be a valid audit action/module"Action/module not in the enumsAdd the constant to AuditAction/AuditModule and rebuild
Audit silently not recordedAudit service down — client swallows the error (by design)Check the warning log; verify the Audit service is up on 8081

9. Final checklist

#Item
1Service has its own @SpringBootApplication main class and a unique server.port
2Bucket A (security) present — via civicdesk-common or copied
3app.jwt.secret is byte-for-byte identical to IAM
4SecurityConfig permitAll() list trimmed to your own public paths
5Controllers read the caller via SecurityContextUtil and guard with @PreAuthorize
6Your AuditAction constants added; app.audit.base-url set
7AuditClient.record(...) called after each create/update/delete, forwarding the JWT
8Verified end-to-end: login on 8081 → call your endpoint → audit row appears via GET /audit/auditLogs
