Checkpoint Reference

Checkpoint 1 — CORS

CorsConfig

  • Validates origin before processing
  • Exposes Authorization header for token refresh
Checkpoint 2 — JwtAuthFilter

Token validation

  • Reads Authorization: Bearer <jwt>
  • Invalid/expired → 401
  • Populates SecurityContext with userId + ROLE_*
  • Sliding expiry: refresh when ≤ 10 min left
Checkpoint 3 — URL Authorization

SecurityConfig

  • Stateless sessions, CSRF disabled
  • permitAll: register, logins, setPassword, Swagger
  • All else → authenticated()
Checkpoint 4 — Method RBAC

@PreAuthorize

  • hasRole('ADMIN') — status updates
  • hasAnyRole('ADMIN','DEPT_SUPERVISOR') — user mgmt
  • hasAnyRole('ADMIN','COMPLIANCE_OFFICER') — audit logs
  • Mismatch → 403
Checkpoint 5 — Domain Rules

Service layer

  • Account status (ACTIVE / SUSPENDED)
  • passwordSet first-login enforcement
  • Department-scoped creation rules
Cross-cutting

GlobalExceptionHandler & Audit

  • Maps exceptions → structured ErrorResponse
  • Audit log on REGISTER, LOGIN, LOGOUT, SET_PASSWORD, CREATE_USER, UPDATE_STATUS
