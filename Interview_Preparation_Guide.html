A. Java Core & OOP

Foundation
#ConceptTheory (What is it?)Where Used in Project
1Classes, Objects, ConstructorsA class is a blueprint; an object is its instance; a constructor initializes the object's state.Every layer — User entity, UserService, UserController. Lombok generates constructors (@AllArgsConstructor).Difference between class and object?
2InheritanceA child class reuses fields/methods of a parent class (IS-A relationship).Custom exceptions extend RuntimeException in ResourceNotFoundException.java.
3PolymorphismOne interface, many forms — overriding (runtime) & overloading (compile-time).Service interface reference points to its implementation; overridden toString()/equals() via Lombok.
4AbstractionHiding implementation, exposing only behaviour via interfaces/abstract classes.UserService interface hides logic from UserController; JpaRepository abstracts DB access.
5EncapsulationBinding data + methods together; private fields exposed via getters/setters.All entity/DTO fields are private with Lombok @Getter/@Setter.Why private fields?
6Interfaces & Abstract ClassesInterface = pure contract; abstract class = partial implementation + contract.UserRepository extends JpaRepository; UserService interface implemented by UserServiceImpl.
7Collections (List, Map, Set)Framework of reusable data structures for storing groups of objects.Service returns List<UserDto>; Map used for grouping; Set for @OneToMany child collections.
8Exception Handlingtry/catch/finally handles runtime errors; custom exceptions model domain failures.try/catch in service; custom ResourceNotFoundException thrown and caught in @ControllerAdvice.
9GenericsType-safe parameterized types — catch errors at compile time.JpaRepository<User, Long>, ResponseEntity<UserDto>, List<T>.
10AnnotationsMetadata read by compiler/frameworks to drive behaviour.Everywhere — @Entity, @Service, @GetMapping, @Test.
11Java 8+ (Streams, Lambda, Optional)Functional-style data processing; Optional avoids null checks.Service uses stream().map(this::toDto).collect(); repository.findById() returns Optional.Why Optional over null?

F. Spring Data JPA & Hibernate

Persistence
#ConceptTheory (What is it?)Where Used in Project
1ORM ConceptObject-Relational Mapping maps Java objects ↔ DB tables, avoiding raw SQL.Entities map to MySQL tables; Hibernate generates SQL automatically.
2@Entity, @Table, @Id, @GeneratedValueMark a persistent class, its table, primary key & key-generation strategy.User.java — @Entity @Table(name="users"), @Id @GeneratedValue(IDENTITY).
3@Column & Relationships@Column maps fields; @OneToMany/@ManyToOne/@ManyToMany/@JoinColumn map associations.@ManyToOne from child → parent with @JoinColumn(name="user_id").
4JpaRepository / CrudRepositoryReady-made CRUD methods: save, findAll, findById, deleteById.UserRepository extends JpaRepository<User, Long> — zero SQL written.Difference between the two?
5@Query (JPQL & Native)Custom queries — JPQL (entity-based) or native SQL.Derived queries like findByEmail() + @Query for complex lookups.
6@TransactionalWraps methods in a DB transaction — all-or-nothing commit/rollback.On service methods that modify data (createUser, updateUser).
7Hibernate as JPA ProviderJPA is the spec; Hibernate is the default implementation behind Spring Data JPA.Hibernate generates & executes SQL under the hood.
8Lazy vs Eager LoadingLazy loads related data on access; eager loads immediately with the parent.@ManyToOne(fetch=LAZY) chosen to avoid the N+1 problem.What is the N+1 problem?
9Cascade TypesPropagate operations (PERSIST, MERGE, REMOVE, ALL) from parent to children.@OneToMany(cascade=CascadeType.ALL) on parent entity.
10Pagination & SortingPageable/Sort fetch data page-by-page instead of all at once.findAll(PageRequest.of(page,size,Sort.by("name"))) in service.

G. MySQL & ANSI SQL

Database
#ConceptTheory (What is it?)Where Used in Project
1DDL vs DML vs DQLDDL defines schema (CREATE/ALTER/DROP); DML changes data (INSERT/UPDATE/DELETE); DQL reads (SELECT).Hibernate generates DDL from entities; DML/DQL run on CRUD calls.
2CREATE / ALTER / DROP TABLEDefine and modify table structure.Auto-generated from @Entity classes via ddl-auto.
3SELECT / INSERT / UPDATE / DELETECore data manipulation statements.Triggered by findAll, save, deleteById repository methods.
4WHERE, ORDER BY, GROUP BY, HAVINGFilter, sort, aggregate & filter aggregates.Generated for derived queries (findByStatusOrderByName) & reports.
5JOINs (INNER / LEFT / RIGHT)Combine rows from multiple tables on related columns.Hibernate emits JOINs across mapped relationships (user ↔ orders).INNER vs LEFT join?
6Primary Key, Foreign Key, ConstraintsPK uniquely identifies rows; FK links tables; constraints enforce integrity.@Id → PK; @JoinColumn → FK; @Column(unique,nullable) → constraints.
7IndexesSpeed up lookups on frequently-queried columns.Index on email via @Column(unique=true) / @Index.
8Transactions (COMMIT / ROLLBACK)Group statements so they all succeed or all fail (ACID).Managed by @Transactional at the service layer.
9Stored Procedures (concept)Pre-compiled SQL routines stored in the DB.Concept-aware; can be called via @Procedure if needed.
10JPA Auto DDLspring.jpa.hibernate.ddl-auto = none/validate/update/create/create-drop.Set to update in application.properties during development.Why never use create-drop in prod?

K. Design Patterns & Principles

Architecture
#ConceptTheory (What is it?)Where Used in Project
1SOLID — S (Single Responsibility)A class should have one reason to change.Controller handles HTTP, service handles logic, repo handles data — separate roles.
2SOLID — O (Open/Closed)Open for extension, closed for modification.New service implementations added without changing the interface.
3SOLID — L (Liskov Substitution)Subtypes must be replaceable for their base type.UserServiceImpl safely substitutes the UserService interface.
4SOLID — I (Interface Segregation)Prefer small, specific interfaces over fat ones.Focused service/repository interfaces, no unused methods.
5SOLID — D (Dependency Inversion)Depend on abstractions, not concrete classes.Controller depends on UserService interface, not the impl.How does DI enable DIP?
6SingletonOne shared instance for the whole app.Every Spring bean is a singleton by default.
7BuilderStep-by-step construction of complex objects.Lombok @Builder on DTOs/entities.
8Repository PatternAbstracts data access behind a collection-like interface.UserRepository via Spring Data JPA.
9DTO PatternTransfer objects separate API model from persistence model.Request/Response DTOs in the controller layer.
10Factory (concept)Centralizes object creation logic.Spring's BeanFactory creates beans; aware as a concept.
11MVC PatternModel–View–Controller separates data, presentation & control.Spring MVC: Controller → Service → Repository, JSON as the "view".
